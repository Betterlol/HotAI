### 远程部署流程

#### 1. 本地开发

#### 2. 本地部署和测试

```bash
# 只有前段改动的话
docker compose -f docker-compose.dev.yml up -d
cd web/default
bun install
bun run dev
```

```bash
# 有后端改动的话（Go 代码改动），需要重新 build new-api 镜像
docker compose -f docker-compose.dev.yml up -d --build new-api
```

#### 3. 远程部署

在本地开发和测试完成后，确认代码没有问题后，就可以进行远程部署了。

##### 3.1 构建docker镜像

```bash
# docker 镜像的 tag 名
export APP_IMAGE=ghcr.io/betterlol/hotai:版本号
# 呃，建议使用 git tag 来管理版本号，或者使用 git commit hash 来管理版本号
# export APP_IMAGE=ghcr.io/betterlol/hotai:$(git rev-parse --short HEAD)
# 构建docker镜像
docker compose -f docker-compose.build.yml build
```

##### 3.2 推送docker镜像到远程仓库

```bash
docker compose -f docker-compose.build.yml push
```

##### 3.3 拉取远程仓库的镜像并启动服务

```bash
# 登陆远程服务器后，cd 到 /home/admin/newapi 目录下，修改 .env 文件中的 APP_IMAGE 为你刚刚构建的镜像名后，执行如下命令：
docker compose pull
docker compose up -d
```
