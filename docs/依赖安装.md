  路线 A：最简单（SQLite，无需 Docker）

  1. 安装依赖

  Go 1.25+
  # Linux/macOS（用官方脚本）
  wget https://go.dev/dl/go1.25.1.linux-amd64.tar.gz
  sudo tar -C /usr/local -xzf go1.25.1.linux-amd64.tar.gz
  echo 'export PATH=$PATH:/usr/local/go/bin' >> ~/.bashrc
  source ~/.bashrc
  go version

  Bun（前端包管理器）
  curl -fsSL https://bun.sh/install | bash
  source ~/.bashrc
  bun --version

  2. 安装前端依赖并构建

  cd /home/yihan/HotAI/web/default
  bun install
  bun run build

  构建产物会输出到 web/default/dist/，Go 编译时会通过 //go:embed 打包进二进制。

  3. 编译并运行后端

  cd /home/yihan/HotAI
  go build -o new-api .
  ./new-api

  默认监听 http://localhost:3000，首次启动会自动创建 SQLite 数据库文件（/data/new-api.db 或当前目录）。初始 root 密码会打印在终端日志里。

  ---
  路线 B：开发模式（前后端分离，PostgreSQL + Redis）

  适合频繁改代码，前端有热更新，后端不用重新打包前端。

  1. 额外依赖：Docker + Docker Compose

  # Ubuntu/Debian
  sudo apt-get update
  sudo apt-get install -y docker.io docker-compose-plugin

  # 将当前用户加入 docker 组（免 sudo）
  sudo usermod -aG docker $USER
  newgrp docker

  2. 启动数据库和 Redis

  cd /home/yihan/HotAI

  # 构建后端镜像（跳过前端，用占位 HTML）并启动 PostgreSQL + Redis
  docker compose -f docker-compose.dev.yml up -d --build

  第一次会拉取镜像并编译 Go，稍等片刻。完成后后端运行在 localhost:3000。

  验证是否正常：
  docker compose -f docker-compose.dev.yml logs -f new-api

  看到类似 Server started on :3000 即成功。

  3. 启动前端 dev server

  另开一个终端：
  cd /home/yihan/HotAI/web/default
  bun install
  bun run dev

  前端运行在 http://localhost:3001，API 请求自动代理到 :3000。

  4. 后续开发流程

  - 改了 Go 代码：重新构建后端镜像
  docker compose -f docker-compose.dev.yml up -d --build new-api
  - 改了前端代码：Rsbuild 自动热更新，无需操作
  - 停止所有服务：
  docker compose -f docker-compose.dev.yml down
  - 清空数据重来：
  docker compose -f docker-compose.dev.yml down -v

  ---
  路线 C：完整 Docker Compose（最接近生产）

  cd /home/yihan/HotAI
  cp .env.example .env
  # 按需修改 .env 里的密码（SESSION_SECRET 建议改成随机字符串）

  docker compose up -d

  访问 http://localhost:3000，首次进入会有初始化向导。
